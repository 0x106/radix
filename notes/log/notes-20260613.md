# Radix — State of the project (2026-06-13)

This log builds on `notes-20260609.md`. Today's work is a hardening pass: we stopped treating
the runtime as a throwaway discovery artifact and started treating it as the thing we ship.
The new canonical overview is `notes/radix.md` — written as if this design was always the
intention. From here on there is no "Phase 0 / Phase 1 / the real thing comes later" framing;
this _is_ the runtime, and we harden it in place.

All work landed on a `refactor` branch.

## What changed

### 1. The library was restructured and renamed

`contract.ts` was an odd name for what is just the runtime's public type surface, and the
files were all piled under `src/lib/`. New layout:

- `src/runtime/types/` — the type surface, split by foundation: `db.ts` (data), `world.ts`
  (clock/events/random/log/services/actor/stub), `index.ts` (the `RadixRuntime` root +
  re-exports).
- `src/runtime/source.ts` — the inlined browser-JS runtime (was `runtimeSource.ts`).
- `src/runtime/packaging.ts` — HTML packaging (was `htmlTemplate.ts`).
- `src/publish/` — the InstantDB tooling (`instant.ts` was `adminDb.ts`, plus `pushProject.ts`).

The "PHASE 0 / THROWAWAY" headers and the scattered `plan.md Phase N` / `notes.md §N` comment
references are gone.

### 2. Security: the postMessage bridge is locked down

The bridge used to accept a `message` from any window and broadcast responses — including a
full `db.dump()` of user data — to `'*'`. It now only trusts the **direct parent window**,
can be pinned to an exact shell origin (`wrapPrototype({ shellOrigin })`), replies to the
sender at its exact origin, and pushes unsolicited state only to the parent it learned from
the first accepted message. Packaging also escapes any `</script>` in inlined source, and
there's now an `app/.env.example` documenting the (gitignored) secrets.

**Action for Jordan:** rotate the Instant admin token and the OpenAI key — both live values
were visible in `app/.env` during this review. They were never committed, but rotate anyway.

### 3. Correctness: the silent failures now speak up

- Timers, `Math.random`, and `Date` are shadowed in the prototype's scope so an app that
  reaches for the real globals can't silently break replay. This closes the long-standing
  "we can't safely override Math.random globally" worry — the shadowing is lexical and never
  touches React or `window.*`.
- `db.update` on a missing id no longer fabricates a row (throws in strict, warns otherwise).
- Querying an undeclared collection, an unknown where/order field, or an `include` whose
  foreign key resolves to nothing now warns (or throws in strict) instead of returning empty
  / embedding `undefined`.
- A throwing subscriber can no longer break the `subscribe()` call.

### 4. Performance

- The clock's timer queue is kept sorted on insert, so advancing no longer re-sorts the whole
  queue on every iteration — this was quadratic with hundreds of timers (towerdefence,
  spreadsheet).
- IndexedDB write-through is coalesced: a burst of writes flushes once per touched collection
  instead of serializing the whole collection on every single mutation.
- The log is a ring buffer (capped at 2000), and log pushes to the shell are throttled to one
  per 100ms instead of posting the full array on every line.
- Only one real-time clock driver runs at a time (the rAF frame loop replaces the 100ms
  interval while there are `onFrame` subscribers).

### 5. A real test harness — and a bug it caught

There was no automated test net; examples were only ever pushed to a live account. There's now
`npm test` (`src/tests/runtime.test.ts`): it instantiates the runtime headless in Node and
asserts determinism, reset-replay, immutability, query semantics, strict validation, the
locked-down bridge, and the determinism scope — 46 checks. Part of it compiles every example
under the packaging wrapper, which immediately surfaced that **`cron.ts` had an unbalanced
paren and never parsed** — it would have failed in the browser. Fixed.

## Where things stand

The runtime now has a security model, a test net, and a clean module layout, and it no longer
describes itself as disposable. The example apps all still compile. Next worth considering:
deciding the IndexedDB hydration-vs-seed reconciliation story more rigorously (the `touched`
flag is a heuristic), and whether the `where` vocabulary needs more than equality + `in`
before more example apps push on it.
